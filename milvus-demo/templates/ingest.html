逐行写入

适合快速验证 embedding、去重和检索链路。

每行算一条独立记录，系统自动分配 ID，并同步追加到 data/user_data.json 做本地备份。

批量上传文件

支持 JSON / CSV，上传后返回成功、已存在、失败统计。
拖拽 JSON / CSV 文件到这里也可以导入。
未选择文件

如果系统文件框被浏览器或扩展拦截，可以直接拖拽文件到上方区域。

支持 JSON（[{"text":"..."}]）或 CSV（首行含 text 列）。 text 必填，可选字段包括 document_id、category、tags、source。

数据存储说明
  1. 文本会先进入 embedding 模型，再写入当前向量库用于语义检索。
  2. 原始文本同时追加保存到 data/user_data.json，作为本地持久化备份。
命令行导入

准备好 [{"id": 1, "text": "...", "category": "..."}] 格式的 JSON 后，可执行 python -m src.ingest /path/to/your_data.json 或 make ingest-large。

危险操作

清空当前向量库全部数据，不可恢复。
